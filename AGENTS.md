# AGENTS.md

## Project Rules
This is a Unity project. Follow the existing project structure and keep changes minimal and safe.

## Source of Truth
- Read `README.md` first.
- Treat `README.md` as the single source of truth for gameplay scope and MVP requirements.

## Implementation Rules
- Work only within MVP scope.
- Do not add extra systems, polish, content, or architecture beyond what is required for MVP.
- Prefer the simplest implementation that works.
- Make small, verifiable steps.
- After each logical step, explain what was changed.
- If a task is unclear, propose 2–3 simple options and choose the simplest one.

## Unity-Specific Rules
- Do not change project settings unless necessary for the requested task.
- Do not delete or rename assets/scenes/scripts unless required.
- Keep scripts simple and readable.
- Avoid overengineering, patterns, or abstractions unless clearly needed.
- Preserve `.meta` files.

## Git Rules
- Make a commit after each logical completed step.
- Use clear commit messages.
- Do not rewrite history.

## Communication Style
- Before implementation, briefly state the plan.
- After implementation, list:
  - files changed
  - what was done
  - how to verify in Unity
